# Verethier Frontend

Angular verification app for Discord users following `/verify/:data` links.

## Stack

- Angular 21
- TypeScript 5.9
- SCSS
- wagmi 3 + Web3Modal + viem

## What It Does

- Opens verification links generated by the Discord bot
- Connects wallets through WalletConnect, injected wallets, and Coinbase Wallet
- Signs EIP-712 verification payloads
- Sends verification requests to the backend API

## Requirements

- Node.js `20.19+` or `22.12+`
- Yarn `1.22.x`

## Environment Files

- [src/env/env.ts](src/env/env.ts): production/default values
- [src/env/env.dev.ts](src/env/env.dev.ts): used by the Angular development build via file replacement

The checked-in development config currently points at:

```ts
apiUrl: 'http://localhost:3200/api'
```

So either:
- run the backend locally on port `3200`, or
- update [env.dev.ts](src/env/env.dev.ts) for your local backend URL

## Local Development

Install dependencies:

```bash
yarn install
```

Run the Angular dev server:

```bash
yarn ng serve
```

Then open `http://localhost:4200`.

## Scripts

```bash
yarn ng serve       # Angular dev server (default: localhost:4200)
yarn build          # Production build to dist/frontend
yarn watch          # Development build in watch mode
yarn start          # Serve dist/frontend/browser on $PORT
yarn test           # Karma test runner
```

For a headless local/CI-style test run:

```bash
yarn test --watch=false --browsers=ChromeHeadless
```

## Deployment

This app is deployed as a built static bundle plus a simple static file server:

```bash
yarn build
yarn start
```

Notes:
- `yarn start` expects `dist/frontend/browser` to already exist.
- On Railway, prefer `Railpack` instead of deprecated `Nixpacks`.
